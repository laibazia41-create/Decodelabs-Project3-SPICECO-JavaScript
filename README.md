# SPICECO - Client-Side Interactive Logic (Project 3)

Developed during the Industrial Training Engagement Phase with **DecodeLabs** (Batch: 2026).

## 🚀 Project Overview
This project transforms the **SPICECO** interface into a live, reactive web application. By building an internal programmatic logic layer using vanilla JavaScript, the web document captures live client triggers and performs real-time updates to user interface nodes without forcing an application page reload.

## 🛠️ Tech Stack Used
- **HTML5 & CSS3:** Responsive structural components.
- **JavaScript (Vanilla JS):** Event dispatch handlers, state management, and DOM tracking.

## 📐 Key Specifications Implemented
- **The IPO (Input-Process-Output) Loop:** Wired layout monitors that track explicit user interaction events (Input), evaluate internal code states (Process), and apply instant display mutations (Output).
- **Dynamic Shopping Cart Tracker:** Tied customized click listeners to card buttons using `js-` target hooks, calculating and incrementing navigation header numbers instantly via `.textContent`.
- **Dynamic Catalog Filters:** Applied matching data attributes (`data-filter` and `data-category`) to run group classification cycles, hiding or displaying product sections cleanly using standard script logic.
- **Overlaid Modal Viewboxes:** Built a responsive product showcase overlay template, using JavaScript logic to swap hidden style values (`display: none` to `display: flex`) on demand.

## 📁 File Structure
- `index.html` - Living DOM root layout.
- `web.js` - Complete event tracking script logic.
- `CSS/master3.css` - Visual layer and modal popup styles.
